Why 95% of Published Trading Signals Are False

Test 100 strategies on pure noise and 5 will look significant by chance. Most quant researchers publish one of those 5 without knowing it. Multiple testing corrections and the Deflated Sharpe Ratio are your defences вАФ with Python proof.

In Day 6 we learned that a single hypothesis test at ќ±=0.05 has a 5% chance of giving a false positive. Now imagine running 100 tests. Suddenly you expect 5 false positives just by random chance вАФ and most of them will look like genuine discoveries. This is the multiple testing problem and it is the primary reason most published trading signals fail in live trading.

The Core Problem вАФ Illustrated

The code below tests 100 completely random strategies on pure noise вАФ zero real edge in any of them. Watch how many appear significant.

python вАФ 100 zero-edge strategies
import numpy as np
from scipy import stats

np.random.seed(42)

n_strategies = 100
n_obs        = 252   # 1 year daily returns

# ALL strategies = pure noise, zero real edge
all_returns = [np.random.normal(0, 1, n_obs) for _ in range(n_strategies)]
p_values    = [stats.ttest_1samp(r, 0)[1] for r in all_returns]

false_pos = sum(p < 0.05 for p in p_values)
print(f"Strategies tested:          {n_strategies}")
print(f"Strategies with real edge:  0 (all pure noise)")
print(f"False positives (p<0.05):   {false_pos}")
print(f"Expected false positives:   {n_strategies * 0.05:.0f}")

What just happened

100 strategies. Zero real edge. 4 false positives вАФ exactly as the math predicts. A researcher testing these strategies naively would publish 4 "discoveries" and present them at conferences. Funds would allocate capital to them. They would fail in live trading. This is not hypothetical вАФ it is how most quant research works.

Three Methods to Fix the Problem

No correction
Naive p < 0.05

Use the standard threshold without adjusting for multiple tests. Works fine for a single test. Completely breaks down when testing many strategies simultaneously.

Result: 4 false positives вЬЧ
Conservative fix
Bonferroni

Divide ќ± by number of tests. Simple, effective, but very conservative вАФ rejects too many real signals. Best when false positives are extremely costly.

Threshold: 0.05/100 = 0.0005
Balanced fix
Benjamini-Hochberg

Controls the False Discovery Rate вАФ the expected proportion of false positives among all significant results. Less conservative than Bonferroni. Industry standard.

Result: 0 false positives вЬУ

Python вАФ All Three Methods Side by Side

python вАФ Bonferroni + Benjamini-Hochberg
# вФАвФА Bonferroni correction вФАвФА
bonf_threshold = 0.05 / n_strategies   # 0.0005
bonf_sig = sum(p < bonf_threshold for p in p_values)
print(f"Bonferroni threshold:       {bonf_threshold:.4f}")
print(f"Significant after Bonf:     {bonf_sig}")

# вФАвФА Benjamini-Hochberg FDR вФАвФА
def benjamini_hochberg(p_vals, alpha=0.05):
    n = len(p_vals)
    sorted_idx = np.argsort(p_vals)
    sorted_p   = np.array(p_vals)[sorted_idx]
    thresholds = (np.arange(1, n+1) / n) * alpha
    significant = sorted_p <= thresholds
    if not any(significant):
        return np.zeros(n, dtype=bool), 0
    k = np.where(significant)[0][-1]
    reject = np.zeros(n, dtype=bool)
    reject[sorted_idx[:k+1]] = True
    return reject, thresholds[k]

reject_bh, bh_thresh = benjamini_hochberg(p_values)
print(f"BH significant results:     {reject_bh.sum()}")
print(f"Expected FDR вЙ§ 5% of those: {reject_bh.sum()*0.05:.2f}")

Both corrections eliminate all 4 false positives from 100 noise strategies. In practice on real data with genuine signals, Benjamini-Hochberg is preferred because Bonferroni is too conservative вАФ it rejects real discoveries along with fake ones.

The Deflated Sharpe Ratio вАФ The Most Honest Backtest Metric

Even if your strategy passes individual hypothesis tests, there is a deeper problem. The more strategies you tested to find this one, the more likely even a "significant" result is noise. The Deflated Sharpe Ratio (DSR) by Bailey & Lopez de Prado (2014) corrects for this directly.

Deflated Sharpe Ratio вАФ Key Insight
Expected max Sharpe from noise:
E[max SR] = (1вИТќ≥)¬Јќ¶вБї¬є(1вИТ1/N) + ќ≥¬Јќ¶вБї¬є(1вИТ1/Ne)
N = number of strategies tried  ¬Ј  ќ≥ = Euler-Mascheroni constant вЙИ 0.5772
ќ¶вБї¬є = inverse normal CDF

If your observed Sharpe вЙ§ E[max SR from noise] вЖТ your strategy is not beating random luck
PSR = P(true SR > benchmark) вАФ should be > 0.95 to be credible
python вАФ Deflated Sharpe Ratio
from scipy import stats
import numpy as np

sharpe_obs = 1.5    # observed Sharpe after testing 100 strategies
n_trials   = 100   # number of strategies tested
n_obs      = 252   # observations per strategy (1 year daily)

# Euler-Mascheroni constant
gamma = 0.5772156649

# Expected maximum Sharpe from pure noise
e_max_sr = (((1 - gamma) * stats.norm.ppf(1 - 1/n_trials)) +
            (gamma * stats.norm.ppf(1 - 1/(n_trials * np.e))))

# Probabilistic Sharpe Ratio
psr = stats.norm.cdf(
    (sharpe_obs - e_max_sr) * np.sqrt(n_obs - 1) /
    np.sqrt(1 + sharpe_obs**2 / 4)
)

print(f"Observed Sharpe:        {sharpe_obs}")
print(f"Trials run:             {n_trials}")
print(f"Expected max from noise:{e_max_sr:.4f}")
print(f"PSR:                    {psr:.4f}")
print(f"DSR significant:        {psr > 0.95}")

The brutal insight from this output

Observed Sharpe = 1.5 вАФ sounds impressive. But after testing 100 strategies, pure random noise would be expected to produce a maximum Sharpe of 2.5306. Your 1.5 is below what luck alone would give you. PSR = 0.0000. This strategy is not beating chance. Most traders would have traded it. The DSR says it is noise.

Honest vs Dishonest Backtesting

вЭМ How most researchers backtest

Test 100+ parameter combinations.
Pick the one with highest Sharpe.
Report only that result.
Claim p < 0.05 as proof.
Never mention how many tests were run.

Result: False discovery that fails in live trading.

вЬУ Honest backtesting

Report how many strategies were tested.
Apply Bonferroni or BH correction.
Compute Deflated Sharpe Ratio.
Use out-of-sample validation.
PSR > 0.95 before allocating capital.

Result: Far fewer "discoveries" вАФ but real ones.

Three things to remember from Day 8

Multiple testing is the silent killer of quantitative research. Every additional strategy you test inflates your false positive rate вАФ and most researchers never account for this at all.

  • Testing 100 zero-edge strategies produced 4 false positives вАФ exactly as predicted. Most published quant signals are one of these. Always ask: how many strategies did the researcher test before finding this one?
  • Bonferroni divides ќ± by the number of tests (0.05/100 = 0.0005). Benjamini-Hochberg controls the False Discovery Rate more intelligently. Both eliminated all 4 false positives from our 100 noise strategies.
  • Observed Sharpe = 1.5 after 100 trials is worthless вАФ pure noise produces 2.53 on average. The Deflated Sharpe Ratio reveals this. PSR = 0.0000. Always compute DSR before trusting any backtest result.
