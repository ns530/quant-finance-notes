How to Read Market Memory

ACF and PACF tell you whether today's return is related to yesterday's — and how many lags back that relationship extends. They are the essential diagnostic tools before building any ARIMA or time series model.

If you know that large positive returns today tend to be followed by large positive returns tomorrow, that is autocorrelation — and it is tradeable edge. ACF measures this relationship at every lag. PACF isolates each lag's direct contribution. Together they tell you exactly what kind of model your data needs.

What Is Autocorrelation?

Autocorrelation at lag k measures the correlation between a time series and a lagged version of itself. Formally:

ACF formula

ACF(k) = Cov(rₜ, rₜ₋ₖ) / Var(rₜ)

ACF(0) = 1 always (series correlated with itself)
ACF(1) = correlation between today and yesterday
ACF(k) = correlation between today and k days ago
95% CI = ±1.96/√n  →  ±0.1235 for n=252

AR(1) vs White Noise — The Key Contrast

Autocorrelated process
AR(1) with φ=0.4

Each return is 40% correlated with the previous one. ACF decays gradually — significant at lag 1, still visible at lag 2. This is the pattern you want to detect and model.

ACF lag 1: 0.3408 ← significant
ACF lag 2: 0.1041
ACF lag 3: 0.0326
ACF lag 5: 0.0937
No autocorrelation
White Noise

Pure random returns — no memory, no pattern. All ACF values near zero and within the 95% confidence interval. Nothing to model. No predictable structure.

ACF lag 1: 0.0342 ← within CI
ACF lag 2: 0.0043
ACF lag 3: 0.0086
ACF lag 5: −0.0370

Python — Computing ACF from Scratch

python — ACF manual + using statsmodels
import numpy as np
from statsmodels.tsa.stattools import acf, pacf

np.random.seed(42)
n = 252

# AR(1) process with phi=0.4
phi = 0.4
returns_ar = np.zeros(n)
for t in range(1, n):
    returns_ar[t] = phi * returns_ar[t-1] + np.random.normal(0, 1)

# White noise
white_noise = np.random.normal(0, 1, n)

# ACF with 95% CI
acf_ar, ci_ar = acf(returns_ar, nlags=10, alpha=0.05)
acf_wn, ci_wn = acf(white_noise, nlags=10, alpha=0.05)

# 95% CI approximation (Bartlett)
ci_bound = 1.96 / np.sqrt(n)
print(f"95% CI: ±{ci_bound:.4f}")
print()
print("Lag  AR(1)   Sig?   WhiteNoise  Sig?")
for lag in [1,2,3,4,5]:
    ar_sig = "✗ sig" if abs(acf_ar[lag]) > ci_bound else "✓"
    wn_sig = "✗ sig" if abs(acf_wn[lag]) > ci_bound else "✓"
    print(f"{lag:3d}  {acf_ar[lag]:6.4f}  {ar_sig:6}  {acf_wn[lag]:10.4f}  {wn_sig}")

PACF — Partial Autocorrelation

ACF — Total correlation

Correlation between rₜ and rₜ₋ₖ including indirect effects through intermediate lags. Lag 2 ACF includes lag 1 influence. Can be hard to interpret which lag matters directly.

Use to identify MA order (q)
PACF — Direct correlation only

Correlation between rₜ and rₜ₋ₖ after removing all intermediate lag effects. Shows the direct relationship at each lag in isolation. Cleaner signal for model order selection.

Use to identify AR order (p)
python — PACF for AR(1)
from statsmodels.tsa.stattools import pacf

pacf_ar = pacf(returns_ar, nlags=5, method='yw')

print("PACF values for AR(1) process:")
for lag in range(1, 6):
    sig = "← significant (AR order = 1)" if lag == 1 else \
          "← drops off" if abs(pacf_ar[lag]) < 0.12 else ""
    print(f"PACF lag {lag}: {pacf_ar[lag]:.4f}  {sig}")

The key insight from PACF

PACF is significant at lag 1 (0.3408) and drops to near zero immediately after. This is the classic signature of an AR(1) process. The PACF cuts off after lag 1 — telling you to use exactly 1 autoregressive term in your model. ACF alone can't tell you this as cleanly.

Ljung-Box Test — Formal Autocorrelation Test

Instead of eyeballing the ACF plot, the Ljung-Box test formally tests whether any autocorrelation up to lag k is statistically significant.

python — Ljung-Box test
from statsmodels.stats.diagnostic import acorr_ljungbox

# Test first 10 lags
lb_ar = acorr_ljungbox(returns_ar,  lags=[10], return_df=True)
lb_wn = acorr_ljungbox(white_noise, lags=[10], return_df=True)

print(f"AR(1) — LB stat: {lb_ar['lb_stat'].values[0]:.4f}, p: {lb_ar['lb_pvalue'].values[0]:.6f}")
print(f"Autocorrelated: {lb_ar['lb_pvalue'].values[0] < 0.05}")
print()
print(f"White noise — LB stat: {lb_wn['lb_stat'].values[0]:.4f}, p: {lb_wn['lb_pvalue'].values[0]:.4f}")
print(f"Autocorrelated: {lb_wn['lb_pvalue'].values[0] < 0.05}")
▶ python3 ljung_box.py
AR(1) — LB stat: 38.6463, p: 0.000029
Autocorrelated: True
White noise — LB stat: 7.3978, p: 0.6874
Autocorrelated: False
Ljung-Box interpretation

H₀ = no autocorrelation up to lag k
AR(1): LB=38.65, p=0.000029 → reject H₀ → significant autocorrelation → model it with ARIMA
White noise: LB=7.40, p=0.69 → fail to reject H₀ → no autocorrelation → no model needed

Always run Ljung-Box on your residuals AFTER fitting a model too — if residuals still show autocorrelation, your model is incomplete.

Three things to remember from Day 10

ACF and PACF are not just diagnostic tools — they are the map that tells you which model your data actually needs. Reading them correctly is the difference between fitting the right model and overfitting noise.

  • ACF lag 1 = 0.3408 for AR(1) vs 0.0342 for white noise. The 95% CI is ±0.1235. Only the AR(1) breaks through — the Ljung-Box confirms it: p=0.000029 vs p=0.69 for noise.
  • PACF cuts off after lag 1 for AR(1) — dropping from 0.34 to −0.01 immediately. This is the diagnostic signature telling you the model order is exactly 1. ACF alone cannot give you this clean signal.
  • Always run Ljung-Box on residuals AFTER fitting your model. Residuals should look like white noise (p > 0.05). If they don't, your model is missing structure that the ACF/PACF would reveal.
