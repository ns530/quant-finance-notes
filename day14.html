The Mathematical Heart of Portfolio Risk

A portfolio's risk is not the sum of its parts — it depends entirely on how those parts move together. The covariance matrix is the single object that captures this, and every portfolio optimization technique is built on top of it.

Two assets with identical individual volatility can produce wildly different portfolio risk depending on whether they move together or apart. The covariance matrix captures every pairwise relationship in your portfolio simultaneously — it is the object that makes modern portfolio theory possible.

Building a Covariance Matrix — 4 Assets

We simulate 4 assets with a known true correlation structure, then estimate the covariance matrix from 252 days of simulated returns.

python — simulate correlated assets
import numpy as np

np.random.seed(42)
n = 252

# True correlation structure — Stock A, Stock B, Commodity, Bond
true_corr = np.array([
    [1.00,  0.65,  0.30, -0.20],
    [0.65,  1.00,  0.45, -0.15],
    [0.30,  0.45,  1.00,  0.10],
    [-0.20, -0.15, 0.10,  1.00],
])
true_vols = np.array([1.5, 1.2, 2.0, 0.8])  # daily vol %

# Covariance = outer product of vols × correlation
true_cov = np.outer(true_vols, true_vols) * true_corr

# Generate correlated returns via Cholesky decomposition
L = np.linalg.cholesky(true_cov)
returns = (L @ np.random.normal(0, 1, (4, n))).T

# Estimate covariance from the sample
sample_cov  = np.cov(returns.T)
sample_corr = np.corrcoef(returns.T)

Portfolio Variance — The Formula That Changes Everything

Portfolio Variance Formula
σₚ² = wᵀ Σ w
w = vector of portfolio weights  ·  Σ = covariance matrix
This single matrix expression captures every asset's individual variance AND every pairwise covariance simultaneously
Expand for 2 assets: σₚ² = w₁²σ₁² + w₂²σ₂² + 2w₁w₂σ₁σ₂ρ₁₂

Correlation Changes Everything — Two Direct Comparisons

Same individual volatilities. Same 50/50 weights. Only the correlation differs. Watch what happens to portfolio risk.

High positive correlation
Stock A + Stock B (ρ=0.65)

Both assets tend to move together. When one falls, the other usually falls too. Diversification barely helps here.

Correlation ADDS 27.9% risk
Negative correlation
Stock A + Bond (ρ=−0.20)

When stock falls, bond tends to rise slightly. This offsetting behaviour is genuine diversification — it directly reduces portfolio risk.

Correlation CUTS 7.6% risk
python — correlation's effect on portfolio risk
w2 = np.array([0.5, 0.5])

# Stock A + Stock B (positively correlated)
cov_AB   = sample_cov[:2,:2]
var_AB   = w2 @ cov_AB @ w2
naive_AB = np.sum((w2 * np.sqrt(np.diag(cov_AB)))**2)
print(f"Actual vol: {np.sqrt(var_AB):.4f}  vs  If uncorrelated: {np.sqrt(naive_AB):.4f}")

# Stock A + Bond (negatively correlated)
cov_ABond   = sample_cov[[0,3]][:,[0,3]]
var_ABond   = w2 @ cov_ABond @ w2
naive_ABond = np.sum((w2 * np.sqrt(np.diag(cov_ABond)))**2)
print(f"Actual vol: {np.sqrt(var_ABond):.4f}  vs  If uncorrelated: {np.sqrt(naive_ABond):.4f}")

The core lesson of portfolio theory

Adding a highly correlated asset to your portfolio does NOT diversify — it concentrates risk, adding 27.9% more volatility than the uncorrelated case would suggest. Adding a negatively correlated asset genuinely reduces risk. This is why "24 stocks in tech" is not diversification, but "8 stocks across tech, bonds, and commodities" can be — even with fewer holdings.

Positive Semi-Definiteness — The Hidden Requirement

Not every matrix of numbers can be a valid covariance matrix. A real covariance matrix must be positive semi-definite (PSD) — meaning all its eigenvalues must be non-negative. This is not a technicality; it guarantees that no portfolio can have negative variance, which would be mathematically nonsensical.

python — checking positive semi-definiteness
# Eigenvalues must all be >= 0 for a valid covariance matrix
eigvals = np.linalg.eigvalsh(sample_cov)
print(f"Eigenvalues: {eigvals}")
print(f"All non-negative (PSD): {all(eigvals >= -1e-10)}")

Why this breaks in practice

When you estimate a covariance matrix from real data with more assets than observations (common in large portfolios), the sample matrix can become singular or have near-zero eigenvalues — leading to numerically unstable, sometimes negative portfolio "risk" in downstream optimizations. This is precisely the problem shrinkage estimators solve.

Shrinkage Estimators — Taming Estimation Error

The sample covariance matrix is noisy, especially with limited data or many assets. Shrinkage estimators blend the noisy sample matrix with a stable, structured target — dramatically improving real-world performance.

python — simple shrinkage estimator
# Shrink sample covariance toward a diagonal target
target    = np.eye(4) * np.trace(sample_cov) / 4
shrinkage = 0.2   # 20% weight to the stable target

shrunk_cov = (1 - shrinkage) * sample_cov + shrinkage * target

print("Shrunk covariance matrix:")
print(np.round(shrunk_cov, 4))
Why practitioners always shrink

The Ledoit-Wolf shrinkage estimator (the industry-standard version of this idea) optimally chooses the shrinkage intensity to minimise expected estimation error. In practice, shrunk covariance matrices produce portfolio weights that are far more stable out-of-sample than raw sample covariance — because they reduce the impact of noisy, unreliable correlation estimates between less-liquid or newly listed assets.

Three things to remember from Day 14

Portfolio risk is a matrix problem, not a sum. Every serious portfolio construction technique — from Markowitz to risk parity — starts and ends with the covariance matrix.

  • Portfolio variance = wᵀΣw. This single formula captures individual variances and every pairwise covariance at once. Correlation is not a footnote — it directly determines whether adding an asset helps or hurts your risk.
  • High positive correlation (0.65) added 27.9% more risk than the uncorrelated case would suggest. Negative correlation (−0.20) cut 7.6% of risk. True diversification requires assets that move differently, not just "more assets."
  • A valid covariance matrix must be positive semi-definite — all eigenvalues ≥ 0. With limited data or many assets, sample covariance can become unstable. Shrinkage estimators fix this by blending toward a stable structured target.
