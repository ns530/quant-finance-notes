How Quants Fit Models to Real Market Data

Maximum Likelihood Estimation is the mathematical engine behind fitting any statistical model to real return data. It answers one question: which parameters make your observed data most probable? With Python proof.

In Day 1 we chose distributions. In Day 2 we calculated their moments. But how do you actually fit a distribution to your real return data? That is exactly what Maximum Likelihood Estimation answers — and it is the foundation of almost every model in quantitative finance.

The Core Intuition — Before Any Math

Imagine you observe 252 daily returns from a stock. You want to fit a Normal distribution to them. The Normal distribution has two parameters — mean (μ) and standard deviation (σ). MLE asks: which values of μ and σ make the data you actually observed most probable?

Bad parameter guess
μ = 5%, σ = 0.1%

The model says daily returns should cluster tightly around 5%. But your real data has returns ranging from −4% to +4%. This model makes your observed data extremely unlikely.

Log-likelihood ≈ very negative
MLE optimal parameters
μ = 0.044%, σ = 1.448%

These parameters centre the distribution exactly where your data lives and spread it to match the real volatility. Your observed data becomes as probable as possible under this model.

Log-likelihood = −450.85 (maximum)

The Mathematics of MLE

For a dataset of n independent observations x₁, x₂, ..., xₙ and a distribution with parameters θ, the likelihood function is the probability of observing your exact data under those parameters:

Maximum Likelihood Estimation
L(θ) = ∏ f(xᵢ | θ)   →   maximize over θ
ℓ(θ) = Σ log f(xᵢ | θ)   →   log-likelihood (easier to compute)
Multiplying tiny probabilities causes underflow → we take log and sum instead
log turns products into sums · log is monotonic so maximizing ℓ = maximizing L
MLE solution: θ̂ = argmax ℓ(θ)

MLE for Normal Distribution — Step by Step

1
Write the log-likelihood
For Normal distribution, the log-likelihood of n observations is:
ℓ(μ,σ) = −n/2·log(2πσ²) − (1/2σ²)·Σ(xᵢ−μ)²
2
Take derivatives and set to zero
Differentiate with respect to μ and σ, set both to zero — this gives the MLE solutions analytically:
μ̂ = (1/n)·Σxᵢ = sample mean  ·  σ̂² = (1/n)·Σ(xᵢ−μ̂)²
3
Note: MLE sigma uses n, not n−1
MLE gives a biased estimate of sigma (divides by n). The unbiased estimator divides by n−1. For large samples the difference is tiny, but it matters for small datasets.
MLE: σ̂ = std(ddof=0)  ·  Unbiased: s = std(ddof=1)

Python Code — MLE on Real Return Data

python — MLE for Normal distribution
import numpy as np
from scipy import stats

np.random.seed(42)

# Simulate 1 year of daily returns (252 trading days)
returns = np.random.normal(0.05, 1.5, 252)

# MLE estimates for Normal — analytically solved
mu_mle    = returns.mean()          # MLE mean
sigma_mle = returns.std(ddof=0)    # MLE sigma (biased, divides by n)
sigma_ub  = returns.std(ddof=1)    # Unbiased sigma (divides by n-1)

# Log-likelihood at MLE parameters
log_lik = np.sum(stats.norm.logpdf(returns, mu_mle, sigma_mle))

print(f"MLE mu:         {mu_mle:.4f}%")
print(f"MLE sigma:      {sigma_mle:.4f}%")
print(f"Unbiased sigma: {sigma_ub:.4f}%")
print(f"Log-likelihood: {log_lik:.4f}")

# 95% Confidence interval for mu
se_mu  = sigma_mle / np.sqrt(len(returns))
ci_low  = mu_mle - 1.96 * se_mu
ci_high = mu_mle + 1.96 * se_mu
print(f"95% CI for mu: [{ci_low:.4f}, {ci_high:.4f}]")

The CI tells you something important

MLE mu = 0.044% per day sounds positive. But the 95% CI is [−0.134, 0.223] — it crosses zero. This means with 252 data points we cannot statistically confirm there is any positive edge at all. You need far more data to prove a mean return is real. This is the CLT lesson from Day 4 in action.

MLE for Student-t — Numerical Optimisation

For the Normal distribution, MLE has a clean analytical solution. For Student-t it does not — we must use numerical optimisation. This is where scipy's minimize function comes in.

python — MLE for Student-t (fat tails)
from scipy.stats import t as t_dist
from scipy.optimize import minimize

def neg_loglik_t(params, data):
    # Negative log-likelihood for Student-t (we minimize negative = maximize)
    df, loc, scale = params
    if df <= 2 or scale <= 0:
        return 1e10   # invalid params — return huge penalty
    return -np.sum(t_dist.logpdf(data, df=df, loc=loc, scale=scale))

# Numerical optimisation — start from Normal MLE as initial guess
result = minimize(
    neg_loglik_t,
    x0=[4.0, mu_mle, sigma_mle],   # initial guess: df=4, loc, scale
    args=(returns,),
    method='Nelder-Mead'
)

df_mle, loc_mle, scale_mle = result.x

print(f"MLE df:    {df_mle:.4f}")
print(f"MLE loc:   {loc_mle:.4f}%")
print(f"MLE scale: {scale_mle:.4f}%")
print(f"Log-likelihood: {-result.fun:.4f}")
▶ python3 mle_student_t.py

MLE loc:   0.0288%
MLE scale: 1.3578%

df = 16.75 means the MLE found a distribution with mildly fat tails — not as extreme as the real S&P 500 (df ≈ 3–5) but fatter than Normal (df = ∞). With more years of data on real returns, df would typically estimate much lower — closer to 3–6.

MLE in 5 Lines — Your Reusable Function

python — production-ready MLE fitter
from scipy import stats
import numpy as np

def fit_best_distribution(returns: np.ndarray) -> dict:
    """Fit Normal and Student-t via MLE. Return best fit."""

    # Normal MLE (analytical)
    mu, sigma = returns.mean(), returns.std(ddof=0)
    ll_norm = np.sum(stats.norm.logpdf(returns, mu, sigma))

    # Student-t MLE (numerical)
    df, loc, scale = stats.t.fit(returns)   # scipy has built-in MLE!
    ll_t = np.sum(stats.t.logpdf(returns, df, loc, scale))

    # AIC = 2k - 2*log_likelihood (lower = better, penalises complexity)
    aic_norm = 2*2 - 2*ll_norm   # 2 parameters
    aic_t    = 2*3 - 2*ll_t       # 3 parameters

    best = "Student-t" if aic_t < aic_norm else "Normal"

    return {
        "normal":   {"mu": mu, "sigma": sigma, "ll": ll_norm, "aic": aic_norm},
        "student_t":{"df": df, "loc": loc, "scale": scale, "ll": ll_t, "aic": aic_t},
        "best_fit": best
    }

# Use it on your returns
results = fit_best_distribution(returns)
print(f"Best fit: {results['best_fit']}")
print(f"Normal  AIC: {results['normal']['aic']:.2f}")
print(f"Student AIC: {results['student_t']['aic']:.2f}")
The AIC trick — penalising model complexity

Log-likelihood always improves when you add more parameters. AIC (Akaike Information Criterion) = 2k − 2ℓ penalises extra parameters. Normal has 2 parameters, Student-t has 3. If Student-t still wins on AIC, the extra parameter is genuinely worth it. Lower AIC = better model.

Three things to remember from Day 5

MLE does not assume a distribution — it finds the parameters that make your observed data most probable under whichever distribution you choose. That is a fundamentally different and more honest approach than assuming Normal and moving on.

  • For Normal distribution, MLE has a clean analytical solution: μ̂ = sample mean, σ̂ = population std (ddof=0). For other distributions, numerical optimisation via scipy.optimize is the standard approach.
  • Always compare models using log-likelihood or AIC — never assume a distribution fits without checking. Student-t beat Normal even on simulated Normal data in our test.
  • The 95% CI for our MLE mean estimate was [−0.134, 0.223] — crossing zero. With 252 data points you cannot prove any positive edge exists. MLE quantifies your uncertainty, not just your estimate.
