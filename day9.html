Before Any Time Series Model — Check This First

Stationarity is the first assumption every time series model makes. Violate it and your ARIMA, GARCH, and regression results are meaningless. The ADF test tells you if your series is safe to model — with Python proof.

A stationary time series has a constant mean, constant variance, and autocorrelation that does not change over time. Stock prices are not stationary. Log returns almost always are. This single distinction determines whether your entire time series analysis is valid — or garbage.

Stationary vs Non-Stationary — The Core Difference

Non-Stationary
Stock Prices / Random Walk

Mean and variance drift over time. No fixed level the series returns to. Statistics from one period don't apply to another. Models on non-stationary data produce spurious results.

Mean first half: −7.3324
Mean second half: −5.8460 ← different!
Std first half: 4.5646
Std second half: 4.5056
Stationary
Log Returns

Mean and variance roughly constant over time. Series fluctuates around a fixed level. Statistics from one window apply to another. Time series models work correctly here.

Mean first half: +0.0723
Mean second half: −0.0218 ← similar ✓
Std first half: 0.8990
Std second half: 1.0907 ← similar ✓
Why non-stationarity breaks your models

Running regression on two non-stationary price series almost always finds a significant relationship — even if the series are completely unrelated. This is spurious regression. t-stats look great, R² is high, but the relationship is pure coincidence. This is one of the most common errors in financial research.

The ADF Test — Augmented Dickey-Fuller

The ADF test formally tests for a unit root — the technical term for a random walk / non-stationary series.

ADF Test Hypotheses
H₀: The series has a unit root (non-stationary)
H₁: The series has no unit root (stationary)
ADF statistic more negative → stronger evidence against H₀
p-value < 0.05 → reject H₀ → series IS stationary ✓
p-value > 0.05 → fail to reject H₀ → series is NON-stationary ✗

Critical values (n≈252): −3.46 (1%), −2.87 (5%), −2.57 (10%)
ADF statistic must be MORE NEGATIVE than critical value to reject H₀

Python — ADF Test on Prices vs Returns

python — ADF test
import numpy as np
from statsmodels.tsa.stattools import adfuller

np.random.seed(42)
n = 252

# Non-stationary — random walk (price-like)
random_walk = np.cumsum(np.random.normal(0, 1, n))

# Stationary — log returns
log_returns = np.random.normal(0, 1, n)

def run_adf(series, name):
    r = adfuller(series, autolag='AIC')
    print(f"=== {name} ===")
    print(f"ADF Statistic: {r[0]:.4f}")
    print(f"p-value:       {r[1]:.4f}")
    print(f"Critical 1%:   {r[4]['1%']:.4f}")
    print(f"Critical 5%:   {r[4]['5%']:.4f}")
    print(f"Stationary:    {r[1] < 0.05}")

run_adf(random_walk, "Random Walk (Price-like)")
run_adf(log_returns,  "Log Returns")

ADF + KPSS — Use Both Together

ADF Test

H₀ = non-stationary
Reject H₀ if p < 0.05
Low p → stationary evidence

p < 0.05 → Stationary ✓
KPSS Test

H₀ = stationary
Reject H₀ if p < 0.05
High p → stationary evidence

p > 0.05 → Stationary ✓
python — ADF + KPSS combined check
from statsmodels.tsa.stattools import adfuller, kpss
import warnings
warnings.filterwarnings('ignore')

def stationarity_check(series, name):
    adf   = adfuller(series, autolag='AIC')
    kpss_r = kpss(series, regression='c', nlags='auto')
    adf_ok  = adf[1]   < 0.05   # True = stationary
    kpss_ok = kpss_r[1] > 0.05   # True = stationary
    verdict = "STATIONARY ✓" if adf_ok and kpss_ok else "NON-STATIONARY ✗"
    print(f"{name}:")
    print(f"  ADF  p={adf[1]:.4f}  ({'stationary' if adf_ok else 'not stationary'})")
    print(f"  KPSS p={kpss_r[1]:.4f}  ({'stationary' if kpss_ok else 'not stationary'})")
    print(f"  Verdict: {verdict}")

stationarity_check(random_walk, "Random Walk")
stationarity_check(log_returns,  "Log Returns")

The Fix — One Line of Python

python — transform prices to stationary returns
import numpy as np

# Simulate realistic stock prices
prices = 100 * np.exp(
    np.cumsum(np.random.normal(0.0004, 0.01, 252))
)

# One line fix — log returns (first difference of log prices)
log_returns = np.diff(np.log(prices))

adf_p = adfuller(prices,      autolag='AIC')[1]
adf_r = adfuller(log_returns, autolag='AIC')[1]

print(f"Prices p-value:      {adf_p:.4f}  → {'stationary' if adf_p<0.05 else 'NOT stationary'}")
print(f"Log returns p-value: {adf_r:.6f}  → {'stationary' if adf_r<0.05 else 'NOT stationary'}")

Your Workflow — Before Every Time Series Model

Run this before ARIMA, GARCH, VAR, or any regression
1
Plot the series — Does it drift upward? Does variance grow? Non-stationarity is usually visible before any test.
2
Run ADF test — adfuller(series, autolag='AIC'). p < 0.05 = stationary. Report ADF statistic AND critical values.
3
Run KPSS test — kpss(series, regression='c'). p > 0.05 = stationary. Cross-check with decision table above.
4
If non-stationary — fix it — np.diff(np.log(prices)) converts prices to log returns. Almost always achieves stationarity. Retest after.
5
Only then — build your model — ARIMA, GARCH, VAR all require stationary inputs. This step is not optional, it is the foundation.
Three things to remember from Day 9

Stationarity is not a technicality — it is the foundation every time series model stands on. Skip this check and every model result you produce is potentially meaningless.

  • Stock prices are non-stationary (ADF p=0.38). Log returns are stationary (ADF p=0.000, t-stat=−15.54). Always model returns, never raw prices — this is not preference, it is mathematics.
  • Use ADF and KPSS together — they test opposite null hypotheses. ADF: H₀=non-stationary. KPSS: H₀=stationary. Agreement from both gives far stronger confidence than either alone.
  • The fix is one line: np.diff(np.log(prices)). Log returns are almost always stationary, interpretable as continuously compounded returns, and ready to model immediately.
