Why Volatility Explodes in Clusters

Calm markets stay calm. Turbulent markets stay turbulent. This is volatility clustering — and GARCH is the model built specifically to capture it. Here is how it works, fitted to real simulated data with Python.

Returns themselves have almost no autocorrelation, as Day 10 showed. But squared returns — a proxy for volatility — are strongly autocorrelated. Large moves cluster together. This single observation, first documented by Mandelbrot in 1963, is the entire justification for GARCH models.

Volatility Clustering — The Evidence

If today was a big move, tomorrow is more likely to be a big move too — regardless of direction. We test this directly by computing the ACF of squared returns.

python — ACF of squared returns
import numpy as np

np.random.seed(42)
n = 1000

# Simulate GARCH(1,1) process — omega, alpha, beta control clustering
omega, alpha, beta = 0.05, 0.10, 0.85
returns = np.zeros(n)
sigma2   = np.zeros(n)
sigma2[0] = omega / (1 - alpha - beta)

for t in range(1, n):
    sigma2[t]  = omega + alpha * returns[t-1]**2 + beta * sigma2[t-1]
    returns[t] = np.random.normal(0, np.sqrt(sigma2[t]))

# ACF of squared returns — the volatility clustering signature
sq_returns = returns**2
for k in range(1, 6):
    acf_k = np.corrcoef(sq_returns[:-k], sq_returns[k:])[0,1]
    print(f"ACF of squared returns, lag {k}: {acf_k:.4f}")

On Day 10, white noise ACF at lag 1 was 0.0342 — essentially zero. Here, squared returns show consistent positive autocorrelation of 0.03–0.11 across all 5 lags. Returns are unpredictable. Squared returns are not. This is exactly what GARCH is designed to capture — the size of tomorrow's move depends on today's move, even when direction does not.

The GARCH(1,1) Model — Three Parameters

ω
Omega

The long-run baseline variance level. Determines the "resting" volatility when no shocks are occurring.

True: 0.05 · Fitted: 0.0581
α
Alpha

Reaction to recent shocks. How much yesterday's squared return impacts today's variance. Higher = more reactive.

True: 0.10 · Fitted: 0.0752
β
Beta

Persistence of past variance. How much yesterday's variance carries into today. High beta = long memory.

True: 0.85 · Fitted: 0.8561
GARCH(1,1) Model
σₜ² = ω + α·εₜ₋₁² + β·σₜ₋₁²
σₜ² = today's conditional variance  ·  εₜ₋₁² = yesterday's squared shock
σₜ₋₁² = yesterday's variance  ·  α + β = persistence (must be < 1 for stationarity)
Unconditional variance = ω / (1 − α − β)

Python — Fitting GARCH(1,1) with the arch Package

python — fit GARCH(1,1)
from arch import arch_model

# Fit GARCH(1,1) to the simulated returns
model = arch_model(returns, vol='Garch', p=1, q=1, mean='Zero')
fit = model.fit(disp='off')

print(fit.params)
print(f"Log-likelihood: {fit.loglikelihood:.4f}")
print(f"AIC: {fit.aic:.4f}")

Even with 1,000 observations of noisy simulated data, GARCH recovers parameters remarkably close to the true values — especially β, the persistence parameter, which is often the most important for practical volatility forecasting.

Volatility Persistence — Why It Matters

Persistence = α + β = 0.95

This number tells you how long a volatility shock lingers. Persistence close to 1 means shocks decay very slowly — a volatile week can influence expected volatility for months. Real equity markets often show persistence of 0.90–0.99. Persistence ≥ 1 would mean the process is non-stationary (IGARCH) — variance shocks never fully die out.

Forecasting Future Volatility

forecasts = fit.forecast(horizon=5)
variance_forecast = forecasts.variance.values[-1]

for i, v in enumerate(variance_forecast, 1):
    print(f"Day {i}: variance={v:.4f}, vol={np.sqrt(v):.4f}")

GARCH vs EGARCH — When Direction Matters

GARCH — Symmetric

Treats positive and negative shocks identically. A −2% day and a +2% day increase future variance by exactly the same amount, since the model only sees εₜ₋₁². Simple, robust, works well for many assets.

EGARCH — Asymmetric

Captures the leverage effect — in equity markets, negative shocks typically increase future volatility more than positive shocks of the same size. Models log(σₜ²) directly, so no positivity constraint is needed on parameters.

The leverage effect — why EGARCH matters for stocks

When stock prices fall, financial leverage (debt-to-equity) rises mechanically, increasing perceived risk and future volatility. This means crashes are followed by MORE volatility than equivalent-sized rallies. Standard GARCH cannot capture this asymmetry — EGARCH and GJR-GARCH extensions were built specifically to fix this. Fit both and compare AIC when working with equity data.

Three things to remember from Day 12

Volatility is far more predictable than returns. GARCH exploits this predictability — and the parameters it estimates tell you directly how markets absorb and carry shocks over time.

  • Squared returns show clear autocorrelation (0.03–0.11 across 5 lags) even when raw returns show almost none. This asymmetry — unpredictable direction, predictable magnitude — is the entire foundation of GARCH modelling.
  • Fitted GARCH(1,1) recovered ω=0.058, α=0.075, β=0.856 against true values of 0.05, 0.10, 0.85 — remarkably close given noisy data. Persistence α+β=0.95 means volatility shocks decay slowly and clusters persist for extended periods.
  • For equities, prefer EGARCH or GJR-GARCH over plain GARCH — the leverage effect means crashes increase future volatility more than equivalent rallies. Standard symmetric GARCH misses this entirely.
