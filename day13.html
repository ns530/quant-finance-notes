Modeling Multiple Markets Together

Every model so far predicted one series from its own past. VAR predicts several series from each other's pasts simultaneously — revealing which markets actually lead which, with Python proof.

Stocks, bonds, currencies, and commodities do not move in isolation. A bond market selloff often precedes equity weakness. A currency shock ripples into commodity prices. Vector Autoregression (VAR) captures these cross-market relationships in a single unified framework — and tells you statistically which market moves first.

From AR to VAR — The Extension

Day 11's ARIMA modeled one series using its own past values. VAR extends this to multiple series, where each variable is a function of the past values of ALL variables in the system — including itself.

VAR(1) Model — Two Variables
stockₜ = c₁ + φ₁₁·stockₜ₋₁ + φ₁₂·bondₜ₋₁ + ε₁ₜ
bondₜ  = c₂ + φ₂₁·stockₜ₋₁ + φ₂₂·bondₜ₋₁ + ε₂ₜ
Each variable depends on its OWN lag AND the OTHER variable's lag
φ₁₂ captures how bond's past affects stock's present — the cross-market link
This system can be extended to any number of variables (VAR(p) with k variables)

Python — Simulating and Fitting a VAR(1) System

python — VAR(1) with lead-lag relationship
import numpy as np
from statsmodels.tsa.api import VAR

np.random.seed(42)
n = 300

# Simulate stock and bond with a built-in lead-lag relationship
stock = np.zeros(n)
bond  = np.zeros(n)

for t in range(2, n):
    stock[t] = 0.3*stock[t-1] + 0.2*bond[t-1]  + np.random.normal()
    bond[t]  = 0.4*bond[t-1]  - 0.15*stock[t-1] + np.random.normal()

data = np.column_stack([stock, bond])

# Select optimal lag order
model = VAR(data)
lag_selection = model.select_order(maxlags=5)
print(f"AIC selects lag: {lag_selection.aic}")
print(f"BIC selects lag: {lag_selection.bic}")

# Fit VAR(1)
fit = model.fit(1)
print(fit.params)

Both AIC and BIC independently select lag order 1 — matching our true simulation exactly. The fitted coefficient of bond's lag on stock (0.2174) closely matches the true value we simulated (0.2), confirming VAR correctly recovers cross-market dependencies.

Granger Causality — Who Leads, Who Follows?

"Granger causality" does not mean true causation — it means one series has statistically significant predictive power for another, beyond what the second series' own past provides. This is the key diagnostic for understanding lead-lag relationships between markets.

python — Granger causality both directions
from statsmodels.tsa.stattools import grangercausalitytests

# Does Bond Granger-cause Stock?
result_b2s = grangercausalitytests(data[:, [0,1]], maxlag=2, verbose=False)
for lag in [1, 2]:
    f_stat = result_b2s[lag][0]['ssr_ftest'][0]
    p_val  = result_b2s[lag][0]['ssr_ftest'][1]
    print(f"Bond→Stock lag {lag}: F={f_stat:.4f}, p={p_val:.4f}")

# Does Stock Granger-cause Bond?
result_s2b = grangercausalitytests(data[:, [1,0]], maxlag=2, verbose=False)
for lag in [1, 2]:
    f_stat = result_s2b[lag][0]['ssr_ftest'][0]
    p_val  = result_s2b[lag][0]['ssr_ftest'][1]
    print(f"Stock→Bond lag {lag}: F={f_stat:.4f}, p={p_val:.4f}")

The trading insight

The relationship is asymmetric — bond markets lead stock markets in this system, but not the reverse. This directly matches how our data was simulated: stock depends on bond's lag (coefficient 0.2) while the reverse relationship is much weaker. In real markets, this kind of asymmetric lead-lag structure between asset classes is exactly what cross-asset signal strategies attempt to exploit.

Impulse Response — Tracing a Shock Through the System

The Impulse Response Function (IRF) shows what happens to one variable when the other receives a one-time shock. It traces the shock's effect forward through time.

python — impulse response function
# Trace a shock to bond through to its effect on stock
irf = fit.irf(5)

for h in range(5):
    effect = irf.irfs[h][0,1]   # stock's response to bond shock
    print(f"Horizon {h}: {effect:.4f}")

A one-unit shock to bond has zero immediate effect on stock (horizon 0, since the shock hasn't propagated yet), peaks at horizon 1 (0.2174 — matching the direct lag coefficient), then decays smoothly toward zero by horizon 4. This decay pattern tells you how long a cross-market shock takes to work through the system.

VAR vs VECM — When Series Are Cointegrated

VAR — Stationary series

Use standard VAR when all your series are already stationary (Day 9's ADF test passes on each series individually). Works directly on returns, spreads, or any I(0) series.

VECM — Cointegrated series

Use VECM when series are individually non-stationary (I(1), like raw prices) but a linear combination of them IS stationary — meaning they share a long-run equilibrium relationship. VECM models both the short-run dynamics and the error-correction back to equilibrium.

Why this distinction matters

Running VAR directly on two non-stationary price series (like two cointegrated stock prices) produces the spurious regression problem from Day 9. VECM handles this correctly by explicitly modeling the cointegrating relationship — the "error correction term" — that pulls the series back toward their long-run relationship. This is the exact mathematical foundation behind pairs trading, which we will cover in Layer 4.

Three things to remember from Day 13

VAR extends everything from Days 9–12 into a multivariate world — where markets don't just have their own memory, they influence each other's futures too.

  • VAR correctly recovered the simulated cross-market coefficient — bond's lag effect on stock came out as 0.2174 versus the true 0.2. Both AIC and BIC independently selected the correct lag order of 1.
  • Granger causality revealed an asymmetric relationship: Bond→Stock is highly significant (p=0.0001) while Stock→Bond is not (p=0.482). This asymmetry — one market leading another — is exactly what cross-asset trading signals attempt to exploit.
  • Use VAR for stationary series (returns), but switch to VECM when working with non-stationary but cointegrated series (raw prices). This distinction is the mathematical foundation of every pairs trading and statistical arbitrage strategy.
