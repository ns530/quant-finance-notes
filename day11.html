The Classic Model Every Quant Must Know

ARIMA combines everything from the last 3 days — stationarity, ACF, and PACF — into a single forecasting framework. Here is how to select the right order, fit it correctly, and forecast with honest uncertainty bounds.

ARIMA is not one model — it is three combined into one framework. AutoRegressive (AR) captures how past values predict future ones. Integrated (I) handles non-stationarity through differencing. Moving Average (MA) captures how past forecast errors predict future values. Together, ARIMA(p,d,q) is the workhorse of classical time series forecasting.

The Three Components — p, d, q

p
AutoRegressive order

How many past values directly predict the current value. Identified using PACF from Day 10 — where PACF cuts off.

yₜ depends on yₜ₋₁...yₜ₋ₚ
d
Differencing order

How many times you difference the series to achieve stationarity. Determined using the ADF test from Day 9. Usually d=0 or d=1 for returns.

Δᵈyₜ is stationary
q
Moving Average order

How many past forecast errors influence the current value. Identified using ACF from Day 10 — where ACF cuts off.

yₜ depends on εₜ₋₁...εₜ₋ᵩ

Python — Simulating and Fitting an AR(2) Process

python — fit ARIMA to known AR(2) process
import numpy as np
from statsmodels.tsa.arima.model import ARIMA

np.random.seed(42)
n = 300

# Generate AR(2) process: y_t = 0.5*y_(t-1) - 0.3*y_(t-2) + noise
phi1, phi2 = 0.5, -0.3
y = np.zeros(n)
for t in range(2, n):
    y[t] = phi1*y[t-1] + phi2*y[t-2] + np.random.normal(0, 1)

# Fit ARIMA(2,0,0) — since we know the true order is AR(2)
model = ARIMA(y, order=(2, 0, 0))
fit = model.fit()

print(f"True phi1={phi1}, phi2={phi2}")
print(f"Fitted params: {fit.params}")
print(f"AIC: {fit.aic:.4f}")
print(f"BIC: {fit.bic:.4f}")

Model Selection — Finding the Right Order Automatically

In practice you do not know the true order. You fit multiple candidate models and select the one with the lowest AIC or BIC — balancing fit quality against complexity.

Information Criteria
AIC = 2k − 2·ln(L̂)     BIC = k·ln(n) − 2·ln(L̂)
k = number of parameters  ·  L̂ = maximised likelihood  ·  n = sample size
Lower AIC/BIC = better model  ·  BIC penalises complexity more heavily than AIC
Always compare across multiple (p,d,q) combinations — never guess the order
python — grid search over ARIMA orders
results = []
for p in range(0, 4):
    for q in range(0, 3):
        try:
            m = ARIMA(y, order=(p, 0, q))
            f = m.fit()
            results.append((p, 0, q, f.aic, f.bic))
        except:
            pass

# Sort by AIC — lowest wins
results.sort(key=lambda x: x[3])

print("Order      AIC        BIC")
for r in results[:6]:
    print(f"({r[0]},{r[1]},{r[2]})   {r[3]:10.4f}  {r[4]:10.4f}")

Forecasting with Confidence Intervals

A forecast without uncertainty bounds is dangerously incomplete. ARIMA gives you both the point forecast and the confidence interval that widens as you forecast further into the future.

python — forecast with 95% CI
# Forecast 5 steps ahead with confidence intervals
fc_result = fit.get_forecast(steps=5)
point_forecast = fc_result.predicted_mean
ci = fc_result.conf_int(alpha=0.05)

print("Step  Forecast   95% CI")
for i in range(5):
    print(f"t+{i+1}   {point_forecast[i]:7.4f}   [{ci[i][0]:.4f}, {ci[i][1]:.4f}]")

Why forecasts converge to zero and widen

Notice the point forecast shrinks toward the unconditional mean (near 0) as you forecast further ahead — this is a property of stationary AR processes reverting to their mean. The confidence interval also widens with each step, from ±1.9 at t+1 to ±2.1 at t+5. This reflects genuine uncertainty compounding over the forecast horizon. Any forecast presented without these widening bounds is misleading.

ARIMA in Finance — Honest Limitations

Why ARIMA alone rarely beats a random walk on prices

Financial returns typically have very weak autocorrelation (as we saw in Day 10 — white noise ACF near zero for real markets). This means ARIMA on raw returns often reduces to forecasting close to zero — not much better than a naive random walk forecast. ARIMA becomes genuinely useful for volatility modelling (paired with GARCH, Day 12), for macro series with real structure, or as one input feature among many in an ML pipeline — not as a standalone trading signal.

Three things to remember from Day 11

ARIMA is a systematic 3-day payoff — stationarity from Day 9, ACF/PACF from Day 10, now combined into one forecasting model. Never skip the diagnostic steps that come before fitting.

  • Grid search over (p,d,q) combinations correctly identified ARIMA(2,0,0) as the best model among 12 candidates — matching the true underlying process exactly. Never guess the order; always compare AIC/BIC across a grid.
  • Forecasts revert toward the unconditional mean and confidence intervals widen with horizon — from ±1.9 at t+1 to ±2.1 at t+5. Any point forecast without these bounds is incomplete and potentially dangerous to trade on.
  • ARIMA on raw stock returns often barely beats a naive random walk because real markets have very weak autocorrelation. ARIMA earns its keep in volatility modelling (Day 12: GARCH) and as one feature in broader ML pipelines — not as a standalone price predictor.
