The Foundation of Every Factor Model in Finance

OLS regression gives you alpha, beta, and R-squared from a single equation. It underlies CAPM, Fama-French, and every risk model used by hedge funds. Here is how it works — built from scratch in Python.

Ordinary Least Squares regression is the single most important statistical tool in quantitative finance. It tells you how much of a stock's return is explained by the market, what the stock's excess return is, and how reliable that relationship is. Every factor model — from CAPM to Fama-French — is built on it.

The Three Numbers OLS Gives You

α
Alpha

The intercept. Return the stock generates independent of the market. Positive alpha = outperformance. The holy grail every fund manager claims to have.

Our result: α = 0.0695%/day
β
Beta

The slope. Sensitivity to market moves. β=1.2 means when market rises 1%, the stock rises 1.2%. It is your market risk exposure.

Our result: β = 1.2184
R²
R-Squared

How much of the stock's variance is explained by the market factor. R²=0.68 means 68% of daily moves are driven by market — 32% is stock-specific noise.

Our result: R² = 0.6835

The OLS Equation

OLS Regression Model
r_stock = α + β · r_market + ε
r_stock = stock daily return  ·  r_market = market daily return
α = alpha (intercept)  ·  β = beta (slope)  ·  ε = residual (unexplained)

OLS minimises: Σ εᵢ² = Σ (r_stock - α - β·r_market)²
Solution: β̂ = Cov(r_stock, r_market) / Var(r_market)  ·  α̂ = mean(r_stock) - β̂·mean(r_market)

Python — OLS From Scratch AND Using scipy

python — OLS two ways
import numpy as np
from scipy import stats

np.random.seed(42)
n = 252

# Simulate 1 year: market + stock returns
market = np.random.normal(0.04, 1.0, n)
stock  = 0.05 + 1.2 * market + np.random.normal(0, 0.8, n)

# ── Method 1: OLS from scratch using matrix algebra ──
X = np.column_stack([np.ones(n), market])   # design matrix
beta = np.linalg.inv(X.T @ X) @ X.T @ stock   # (XᵀX)⁻¹Xᵀy
print(f"[Scratch] Alpha: {beta[0]:.4f}  Beta: {beta[1]:.4f}")

# ── Method 2: scipy linregress (easier) ──
slope, intercept, r, p, se = stats.linregress(market, stock)

# R-squared and residuals
y_pred     = intercept + slope * market
residuals  = stock - y_pred
r_squared  = 1 - np.sum(residuals**2) / np.sum((stock - stock.mean())**2)

print(f"[scipy]  Alpha: {intercept:.4f}  Beta: {slope:.4f}")
print(f"R-squared:  {r_squared:.4f}")
print(f"Correlation:{r:.4f}")
print(f"p-value β:  {p:.6f}")

Matrix algebra vs scipy — same answer, different paths

Both methods give identical results: α=0.0695, β=1.2184. The matrix formula (XᵀX)⁻¹Xᵀy is how OLS is derived mathematically — it is the closed-form solution that minimises the sum of squared residuals. scipy.linregress does the same computation more efficiently. For understanding, know the matrix formula. For production, use scipy.

Interpreting the Coefficients — With Significance Tests

python — full coefficient table
# Compute t-stats and p-values for both coefficients
n_params = 2
mse      = np.sum(residuals**2) / (n - n_params)

se_beta  = se   # scipy gives SE of slope directly
se_alpha = np.sqrt(mse * (1/n + market.mean()**2 /
           np.sum((market - market.mean())**2)))

t_alpha = intercept / se_alpha
t_beta  = slope     / se_beta
p_alpha = 2 * (1 - stats.t.cdf(abs(t_alpha), df=n-2))
p_beta  = 2 * (1 - stats.t.cdf(abs(t_beta),  df=n-2))

print(f"{'Coef':8} {'Value':10} {'SE':10} {'t-stat':10} {'p-value':10} {'Sig'}")
print(f"{'Alpha':8} {intercept:10.4f} {se_alpha:10.4f} {t_alpha:10.4f} {p_alpha:10.4f} {'***' if p_alpha<0.05 else 'ns'}")
print(f"{'Beta':8} {slope:10.4f} {se_beta:10.4f} {t_beta:10.4f} {p_beta:10.6f} {'***' if p_beta<0.05 else 'ns'}")

The alpha problem

Alpha = +0.0695%/day looks like outperformance. But p-value = 0.1709 — not statistically significant. We cannot reject the hypothesis that alpha is zero. The beta however is rock solid: p≈0.000, t-stat=23.2. The market factor explains this stock with extremely high confidence. Alpha is the hardest thing to prove in finance — and the most overclaimed.

Residual Diagnostics — Is Your Model Valid?

OLS makes four assumptions. Violating them means your alpha and beta estimates are unreliable. Always check residuals before trusting your regression output.

✓ What good residuals look like

Mean ≈ 0.000000 ✓
Normally distributed ✓
No autocorrelation (Durbin-Watson ≈ 2.0) ✓
Constant variance (homoscedastic) ✓
Our DW = 1.9335 — excellent ✓

✗ Red flags in residuals

Durbin-Watson < 1.5 or > 2.5 → autocorrelation
Residuals fan out → heteroscedasticity
Residuals show pattern → missing variable
Heavy tails in residual histogram → model misspecification

python — residual diagnostics
from scipy import stats

# Durbin-Watson statistic (tests for autocorrelation)
dw = np.sum(np.diff(residuals)**2) / np.sum(residuals**2)

# Normality test on residuals (Shapiro-Wilk)
_, p_normal = stats.shapiro(residuals)

print(f"Mean residual:  {residuals.mean():.6f}  (should be ~0)")
print(f"Std residual:   {residuals.std():.4f}")
print(f"Durbin-Watson:  {dw:.4f}    (2.0 = no autocorrelation)")
print(f"Shapiro p-val:  {p_normal:.4f}  (>0.05 = normally distributed)")

Your Reusable OLS Function

python — production OLS wrapper
import numpy as np
from scipy import stats

def ols_regression(y, x):
    """OLS regression. Returns alpha, beta, R2, and significance."""
    slope, intercept, r, p, se = stats.linregress(x, y)
    y_pred    = intercept + slope * x
    residuals = y - y_pred
    r_squared = 1 - np.sum(residuals**2) / np.sum((y - y.mean())**2)
    dw        = np.sum(np.diff(residuals)**2) / np.sum(residuals**2)
    return {
        "alpha":     round(intercept, 6),
        "beta":      round(slope, 4),
        "r_squared": round(r_squared, 4),
        "p_beta":    round(p, 6),
        "dw":        round(dw, 4),
        "residuals": residuals,
    }

# Use it on any stock vs market
result = ols_regression(stock, market)
print(f"Alpha:     {result['alpha']:.4f}% per day")
print(f"Beta:      {result['beta']}")
print(f"R-squared: {result['r_squared']}")
print(f"p(beta):   {result['p_beta']}")
print(f"DW stat:   {result['dw']}")
Three things to remember from Day 7

OLS is not just a statistics tool. It is the mathematical language of risk decomposition — separating what the market drives from what the stock generates on its own. Every factor model speaks this language.

  • Beta = 1.2184 is statistically rock-solid (p≈0.000, t=23.2). Alpha = 0.0695% looks positive but p=0.17 — not statistically proven. Alpha is the hardest thing to prove in finance and the most overclaimed.
  • R² = 0.6835 means 68% of this stock's daily variance is explained by the market alone. The remaining 32% is idiosyncratic — that is where stock-picking edge, if it exists, lives.
  • Always check residuals. Durbin-Watson near 2.0 confirms no autocorrelation. Shapiro-Wilk p > 0.05 confirms normality. Violated assumptions mean your alpha and beta estimates cannot be trusted.
